# 암호화 알고리즘 종류

## 비대칭키 알고리즘

### 추천 순위 및 선택 가이드

| 순위 | 알고리즘 | 추천 이유 및 특징 | 비고 |
|---|---|---|---|
| 1위 | Ed25519 | 현재 가장 추천되는 방식. 보안성이 매우 높고 키 생성이 즉각적이며, 부채널 공격(side-channel attack)에 강함. | 최신 시스템 권장 |
| 2위 | ECDSA | RSA보다 짧은 키 길이(256비트)로도 강력한 보안을 제공하며 연산 효율이 좋음. | 성능 중시 환경, 그러나 Ed25519 대비 사용할 이유가 없음 |
| 3위 | RSA | 가장 오래된 표준으로 호환성이 매우 뛰어남. 보안을 위해 최소 2048비트, 가급적 4096비트 사용 권장. | 구형 장비 접속용 |

**Ed25519 vs ECDSA**

| 특성 | ECDSA | Ed25519 |
|---|---|---|
| 보안 설계 | 난수 생성 실패 시 위험함 | 난수 영향 없이 안전함 |
| 속도 | 빠름 | 매우 빠름 |
| 호환성 | 우수함 (대부분의 구형 시스템 지원) | 보통 (최신 시스템 위주 지원) |
| 권장 여부 | 호환성이 필요할 때 사용 | 신규 생성 시 최우선 권장 |

### 실무 선택 가이드

- 새로 키를 만든다면: 가장 안전하고 빠른 [Ed25519](https://docs.callgoose.com/general/ed25519-vs-other-keys-for-ssh)를 사용하세요. (명령어: ssh-keygen -t ed25519).
- 오래된 서버에 접속해야 한다면: 호환성 문제로 Ed25519가 작동하지 않을 수 있으므로, [RSA 4096비트](https://www.brandonchecketts.com/archives/ssh-ed25519-key-best-practices-for-2025)를 대안으로 사용하세요. (명령어: ssh-keygen -t rsa -b 4096)
