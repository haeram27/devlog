# Mongodb find query

## Project

### 설정 값

find() 메서드의 두 번째 인자에 필드 포함 여부를 객체 형식으로 전달합니다.

- 기본값 : 프로젝션을 지정하지 않으면 모든 필드를 반환
- 포함(Include): 필드명: 1
- 제외(Exclude): 필드명: 0

### 사용 예시 (MongoDB Shell)

```js
// 'name'과 'age' 필드만 가져오기 (기본적으로 _id는 포함됨)
db.users.find({ status: "A" }, { name: 1, age: 1 })

// _id를 제외하고 'name'만 가져오기
db.users.find({}, { name: 1, _id: 0 })

// 특정 필드만 제외하고 나머지는 다 가져오기
db.users.find({}, { password: 0, internal_code: 0 })
```

### 주의사항

- (중요) 혼합 금지: _id 필드를 제외하고는 포함(1)과 제외(0)를 한 번의 프로젝션에서 섞어서 쓸 수 없습니다. (예: `{ name: 1, age: 0 }`은 에러 발생)

## Sort

find() 쿼리에서 결과를 정렬하려면 sort() 메서드를 연결하여 사용

### 설정 값

- 오름차순 (Ascending): 1
- 내림차순 (Descending): -1

### 사용 예시 (MongoDB Shell)

```js
// 1순위: 'age' 내림차순, 2순위: 'name' 오름차순
db.users.find().sort({ age: -1, name: 1 })
```

## Collation

Collation(콜레이션)은 데이터베이스에서 문자열 데이터를 비교하고 정렬하는 규칙

- 별도 설정이 없으면 `이진(Binary) 비교 방식`을 사용
- Collation을 설정 하면 단순히 바이트 순서로 정렬하는 것이 아니라, 언어별 특성(대소문자 구분, 액센트, 한글 자모음 순서 등)을 고려하여 정렬 결과를 결정

- 용도
  - 대소문자 구분: A와 a를 같은 것으로 볼지, 다른 것으로 볼지 결정합니다.
  - 숫자 정렬: "10"이 "2"보다 뒤에 오게 할지(숫자 인식), 아니면 "1"로 시작하니까 앞에 오게 할지(문자열 인식) 설정합니다.
  - 다국어 지원: 한글의 사전식 순서나 영어의 알파벳 순서 등을 해당 언어의 문화권 규칙에 맞게 처리합니다.

### 주요 설정 항목

- locale: 특정 언어 및 지역 규칙을 선택합니다. (예: ko는 한국어, en은 영어) [2, 3]
- strength: 비교의 세밀함을 결정합니다. (1: 기본 문자만, 2: 액센트 구분, 3: 대소문자까지 구분) [3]
- numericOrdering: true로 설정하면 문자열 내 숫자를 숫자 크기대로 정렬합니다. [2, 3]

### 사용 예시 (MongoDB Shell)

```js
// 한국어 규칙 적용 + 대소문자 구분 없이 정렬
db.users.find().sort({ name: 1 }).collation({ locale: "ko", strength: 2 })

// 문자열 내 숫자를 크기순으로 정렬 (예: "2" < "10")
db.users.find().sort({ version: 1 }).collation({ locale: "en", numericOrdering: true })
```

### 주의사항

- 성능: 정렬이나 검색 시 인덱스에 설정된 Collation과 쿼리의 Collation이 일치해야 인덱스를 효율적으로 사용할 수 있습니다
- 기본값: 별도 설정이 없으면 이진(Binary) 비교 방식을 사용하므로 대소문자를 엄격히 구분합니다.

## Index Hint

쿼리 성능 최적화를 위해 특정 인덱스를 사용하도록 지정하는 방법

find() 쿼리 뒤에 .hint() 메서드를 붙여 사용하며, 인덱스 이름이나 인덱스 필드 구조를 전달합니다.

인덱스 필드 구조를 전달하는 방식이란, 인덱스를 생성할 때 정의했던 필드 구성과 방향(1 또는 -1)을 그대로 전달하면, MongoDB가 해당 구조와 일치하는 인덱스를 내부적으로 찾아내어 사용합니다.

```js
// 1. 인덱스 필드 구조로 지정
db.users.find({ age: { $gt: 20 } }).hint({ age: 1, name: 1 })

// 2. 인덱스 이름으로 지정
db.users.find({ age: { $gt: 20 } }).hint("age_1_name_1")
```

### 주의사항 (양날의 검)

- 유연성 저하: 인덱스 이름을 변경하거나 삭제하면 힌트를 사용한 모든 코드가 에러를 발생시킵니다.
- 오판의 위험: 데이터가 늘어남에 따라 옵티마이저가 더 좋은 경로를 찾을 수도 있는데, 힌트가 이를 방해할 수 있습니다.
- 최후의 수단: 먼저 인덱스 설계를 재검토하고, explain()을 통해 실행 계획을 분석한 뒤에도 해결되지 않을 때 사용하는 것이 좋습니다.
