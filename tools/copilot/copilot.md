# Github Copilot

## Mode

### Ask (질문) 모드

    목적: 코드 변경 없이 정보, 설명, 코드 예시 또는 일반적인 코딩 지식을 얻는 데 사용됩니다.
    사용 시점: 특정 개념을 이해하거나, 라이브러리 사용법을 배우거나, 프로젝트 파일에 직접적인 수정이 필요 없는 질문을 할 때 적합합니다.
    특징: 기존의 대화형 AI 챗봇과 유사하게 작동하며, 코드베이스의 문맥을 활용하여 답변을 제공할 수 있습니다. 

### 2. Edit (편집) 모드

    목적: 사용자가 지정한 특정 코드 블록이나 파일에 대해 작고 정밀한 수정을 적용하는 데 사용됩니다.
    사용 시점: 선택한 코드 리팩토링, 주석 추가, 변수 이름 변경 등 비교적 명확하고 범위가 제한된 변경 작업을 수행할 때 유용합니다.
    특징: 사용자가 여전히 변경 사항에 대한 높은 제어권을 가지며, Copilot은 사용자가 원하는 스타일이나 표준에 맞게 코드를 개선하는 데 집중합니다. 

### 3. Agent (에이전트) 모드

    목적: 여러 파일과 다양한 도구(터미널 명령 실행, 테스트 실행 등)를 사용하여 복잡하고 규모가 큰 작업을 자율적으로 수행합니다.
    사용 시점: 전체 기능 구현, 버그 수정, 여러 파일에 걸친 대규모 리팩토링, 새로운 코드베이스 스캐폴딩 등 다단계 작업이 필요할 때 사용됩니다.
    특징: AI가 스스로 작업 계획을 세우고, 관련 파일을 분석하며, 필요한 수정 작업을 반복적으로 수행하여 목표를 달성합니다. 사용자는 최종 결과물을 검토하고 승인할 수 있습니다. 

### 4. Plan (계획) 모드

    목적: 코드를 작성하기 전에 복잡한 작업을 단계별 계획으로 분해하여 보여줍니다.
    사용 시점: 복잡한 구현 작업을 시작하기 전, 요구사항을 명확히 하고 작업 순서를 시각화하거나 정제할 때 사용됩니다.
    특징: Copilot이 생성한 내부 JSON 계획(plan-{sessionId}.json)을 기반으로 하며, 개발자는 이 계획을 검토하고 필요한 경우 수정하여 실행하기 전에 요구사항을 맞출 수 있습니다.

## Prompt

### `/` (슬래시 명령, 단축 명령)

- 프롬프트를 작성하지 않고도 일반적인 작업을 빠르게 수행할 명령을 호출
- 채팅 입력란에 /를 입력하면 사용 가능한 전체 명령 목록 출력
  - /explain: 활성 편집기의 코드를 설명
  - /fix: 선택한 코드의 문제에 대한 수정을 제안
  - /tests: 선택한 코드에 대한 단위 테스트를 생성
  - /doc: 코드에 대한 문서를 생성
  - /new: 새 프로젝트를 생성
  - /clear: 현재 채팅 세션을 삭제

### `@` (멘션, 컨텍스트)

- 대화에 관련 컨텍스트를 직접 첨부하는 데 사용
- 코드나 파일 내용을 프롬프트에 쉽게 참조하도록 하며, AI 에이전트가 지정 컨텍스트의 내용을 기반으로 답변하도록 함
  - @workspace: 전체 작업 공간의 컨텍스트를 제공하여 AI 에이전트가 workspace 내용을 기반으로 답변
  - @vscode: VS Code 자체의 기능(예: 설정, 키 바인딩 변경 방법 등)에 대한 질의 가능
  - @terminal: 터미널/CLI 관련 질문에 사용됩니다.
  파일/이슈/PR: 특정 파일, 이슈, 또는 Pull Request를 멘션하여 해당 내용에 대한 질의 가능

### `#` (채팅 변수)

- 프롬프트에 특정 정보를 변수 형태로 포함할 때 사용
- 코드나 파일 내용을 프롬프트에 쉽게 참조
  - #selection: 현재 선택된 코드 블록을 참조
  - #file: 특정 파일을 참조
  - #editor: 현재 활성화된 편집기 내용을 참조
