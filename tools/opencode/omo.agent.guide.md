# Oh-My-OpenCode 에이전트 가이드

## 메인 오케스트레이터

### Sisyphus

사용자 요청을 받아 분석·계획·위임·검증까지 전담하는 **최상위 에이전트**.  
서브에이전트로 호출되지 않으며, `task(subagent_type="...")` 목록에도 포함되지 않는다.

| 항목 | 내용 |
|---|---|
| **역할** | 의도 파악 → 탐색·연구 위임 → 구현 위임 → 결과 검증 → 사용자 응답 |
| **핵심 전략** | 병렬 실행(explore·librarian 백그라운드), 전문 에이전트 위임, Oracle 자문 |
| **위임 여부** | **가능** — 적절한 카테고리·서브에이전트에 자유롭게 재위임 |
| **직접 구현** | 단순·명백한 작업에 한해 직접 도구 사용, 그 외에는 위임 우선 |
| **Sisyphus-Junior와의 차이** | Junior는 위임 없이 자신이 직접 실행까지 완료하는 집중 실행기 |

**동작 흐름:**
```
사용자 요청
    → [Sisyphus] 의도 파악 + 탐색/연구 에이전트 병렬 실행
    → 결과 수집 후 구현 계획 수립
    → task(category="...", load_skills=[...])로 Sisyphus-Junior에 위임
           또는 oracle / explore / librarian 등 전문 에이전트 활용
    → 결과 검증 (lsp_diagnostics, 빌드, 테스트)
    → 사용자에게 응답
```

## 서브에이전트 (Subagent Types)

직접 `task(subagent_type="...")` 으로 호출하는 전문 에이전트.

| 에이전트 | 역할 | 주요 사용 시점 | 읽기 | 코드 작성 | 웹 검색 | 분석/추론 | 계획 | 실행 |
|---|---|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **build** | 기본 범용 실행 에이전트. 설정된 권한에 따라 모든 도구 사용 가능 | 직접 구현, 파일 편집, 셸 명령 실행 | ★★★ | ★★★ | ★★ | ★★ | ★ | ★★★ |
| **explore** | 코드베이스 내부 전용 컨텍스트 검색기. "X는 어디에 있는가?" 질문에 특화 | 코드 패턴 발견, 파일 위치 파악, 구조 탐색. 항상 `run_in_background=true`로 병렬 실행 | ★★★ | ✗ | ✗ | ★★ | ✗ | ✗ |
| **librarian** | 외부 참조 전용 검색기. GitHub CLI, Context7, 웹 검색으로 공식 문서·OSS 예시 탐색 | 외부 라이브러리 사용법, 공식 API 문서, 오픈소스 구현 예시. 항상 `run_in_background=true`로 병렬 실행 | ★★ | ✗ | ★★★ | ★★ | ✗ | ✗ |
| **oracle** | 읽기 전용 고지능 추론 전문가. 디버깅과 아키텍처 설계에 특화 | 복잡한 아키텍처 결정, 2회 이상 수정 실패 시 디버깅 자문, 보안·성능 검토 | ★★★ | ✗ | ✗ | ★★★ | ★★★ | ✗ |
| **Metis** | 사전 계획 분석 컨설턴트. 숨겨진 의도·모호성·AI 실패 지점 탐지 | 복잡하고 모호한 요구사항 분석, 구현 전 리스크 파악 | ★★ | ✗ | ✗ | ★★★ | ★★★ | ✗ |
| **Momus** | 작업 계획 검토 전문가. 명확성·검증 가능성·완전성 기준으로 평가 | `.omo/plans/*.md`에 저장된 계획 검토, 구현 전 품질 게이트 | ★★★ | ✗ | ✗ | ★★★ | ★★★ | ✗ |
| **multimodal-looker** | 미디어 파일 분석 전문가. 텍스트로 해석하기 어려운 시각 자료 처리 | PDF, 이미지, 다이어그램에서 정보 추출, 시각 자료 요약 | ★★★ | ✗ | ✗ | ★★★ | ✗ | ✗ |
| **general** | 범용 에이전트. 복잡한 질문 연구와 멀티스텝 태스크 병렬 실행 | 단일 에이전트로 분류하기 어려운 혼합 작업 | ★★★ | ★★ | ★★ | ★★ | ★★ | ★★ |
| **plan** | 계획 전용 에이전트. 편집 도구 일체 사용 불가 | 코드 변경 없이 순수하게 계획만 수립해야 할 때 | ★★★ | ✗ | ✗ | ★★★ | ★★★ | ✗ |
| **Sisyphus-Junior** | 집중 태스크 실행기. Sisyphus와 동일한 원칙, 재위임 없음 | `task(category="...")` 위임 시 내부적으로 실행되는 에이전트 | ★★★ | ★★★ | ★ | ★★ | ★ | ★★★ |

> ★★★ 매우 높음 · ★★ 보통 · ★ 낮음 · ✗ 해당 없음

## 태스크 카테고리 (Task Categories)

`task(category="...")` 호출 시 해당 도메인에 최적화된 모델로 **Sisyphus-Junior**를 실행.

| 카테고리 | 도메인 | 사용 시점 | 읽기 | 코드 작성 | 웹 검색 | 분석/추론 | 계획 | 실행 |
|---|---|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **visual-engineering** | 프론트엔드·UI/UX·디자인 | CSS, 레이아웃, 애니메이션, React 컴포넌트 — **시각 작업은 예외 없이 이 카테고리** | ★★ | ★★★ | ★★ | ★★ | ✗ | ★★★ |
| **artistry** | 창의적·비표준 문제 해결 | 일반 패턴을 벗어난 복잡한 접근이 필요한 작업 | ★★ | ★★★ | ★★ | ★★★ | ★★ | ★★★ |
| **ultrabrain** | 로직 집약적·어려운 문제 | 알고리즘, 복잡한 아키텍처, 수학적 추론 — 목표만 전달, 단계 지시 금지 | ★★★ | ★★★ | ★ | ★★★ | ★★★ | ★★ |
| **deep** | 심층 연구 + 자율 구현 | 연구가 선행돼야 하는 복잡한 구현. **목표 1개 + 결과물 1개**만 전달 | ★★★ | ★★★ | ★★★ | ★★★ | ★★ | ★★★ |
| **quick** | 단순 단일 작업 | 단일 파일 변경, 오타 수정, 명백한 단순 수정 | ★ | ★★ | ✗ | ✗ | ✗ | ★★★ |
| **unspecified-low** | 기타 저노력 | 카테고리 불분명·낮은 노력이 예상되는 작업 | ★ | ★★ | ★ | ★ | ✗ | ★★ |
| **unspecified-high** | 기타 고노력 | 카테고리 불분명·높은 노력이 예상되는 작업 | ★★ | ★★★ | ★★ | ★★ | ★ | ★★★ |
| **writing** | 문서·산문·기술 문서 | README, 가이드, 기술 명세 작성 | ★★ | ✗ | ★★ | ★★ | ✗ | ✗ |

## 에이전트 선택 흐름도

```
사용자 요청
    │
    ├─ 시각/UI/디자인? ──────────────────► visual-engineering
    │
    ├─ 코드베이스 패턴 탐색? ────────────► explore (background)
    │
    ├─ 외부 라이브러리/문서 조회? ───────► librarian (background)
    │
    ├─ 계획 검토/피드백? ────────────────► Momus
    │
    ├─ 복잡한 계획 수립 전 분석? ────────► Metis
    │
    ├─ 2회 이상 실패한 디버깅? ──────────► oracle
    │
    ├─ 로직 집약적 난문? ────────────────► ultrabrain
    │
    ├─ 연구 선행 필요한 구현? ───────────► deep
    │
    ├─ PDF/이미지/다이어그램? ───────────► multimodal-looker
    │
    ├─ 단순 단일 수정? ──────────────────► quick
    │
    └─ 그 외 복잡한 작업 ────────────────► unspecified-high
```

## 병렬 실행 패턴

`explore`와 `librarian`는 항상 백그라운드로 동시에 실행하는 것이 기본 원칙.

```typescript
// 내부 탐색 + 외부 참조를 동시에 실행
task(subagent_type="explore",   run_in_background=true, prompt="...")
task(subagent_type="librarian", run_in_background=true, prompt="...")

// 결과를 기다리지 않고 독립적인 다른 작업 진행
// → 완료 알림(system-reminder) 수신 후 background_output()으로 결과 수집
```
