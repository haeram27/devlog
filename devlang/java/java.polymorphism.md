# Polymorphism (다형성)

자바에서 다형성(Polymorphism)이란 "하나의 객체나 메서드가 여러 가지 형태를 가질 수 있는 능력"을 의미합니다. 쉽게 말해, 사용 방법은 같지만 상황에 따라 결과가 다르게 나타나는 성질입니다.

다형성의 핵심 내용과 구현 방식은 다음과 같습니다.

## 1. 다형성의 핵심 개념

* 참조 변수의 형변환 (업캐스팅): 부모 클래스 타입의 참조 변수로 자식 클래스 타입의 객체를 참조할 수 있습니다. 이를 통해 하나의 변수(부모 타입)로 다양한 실제 객체(자식들)를 통합 관리할 수 있습니다.
* 동일 메시지, 다른 동작: 같은 이름의 메서드를 호출하더라도 실제 참조하고 있는 객체에 따라 서로 다른 실행 결과가 나옵니다.

## 2. 메소드 다형성을 구현하는 방법

자바에서는 크게 두 가지 방식으로 다형성을 나타냅니다.

* 컴파일 타임 다형성 (오버로딩, Overloading): 한 클래스 내에서 메서드 이름은 같지만 매개변수의 타입이나 개수를 다르게 하여 여러 형태로 정의하는 것입니다.
* 런타임 다형성 (오버라이딩, Overriding): 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것입니다. 프로그램 실행 중에 실제 객체의 메서드가 호출되는데, 이를 동적 바인딩이라고 합니다.

## 3. 다형성이 왜 중요한가요?

* 코드의 재사용성: 기존 코드를 수정하지 않고 새로운 클래스를 추가하기 쉽습니다.
* 유연성과 유지보수: 객체 간의 결합도를 낮추어 코드가 유연해지고 관리가 편해집니다.
* 확장성: 새로운 기능을 추가할 때 표준화된 인터페이스(부모 타입)를 사용하므로 확장이 용이합니다. [15, 16, 17, 18, 19]
